# 데이터 출처와 신뢰도

이 문서는 발표·데모 문구가 실제 검증 수준보다 앞서가지 않도록 현재 데이터의 출처와 한계를 구분합니다.

## 발표에서 사용할 표현

### 사용 가능

- "실제 점주가 제공한 자료를 앱 스토리와 코스로 구조화했습니다."
- "실제 장소 상세 화면과 코스 편입 흐름을 구현했습니다."
- "조회·저장·방문 신호를 사장님에게 보여주는 대시보드 프로토타입입니다."

### 검증 전에는 사용하지 말아야 할 표현

- "앱으로 실제 매출이 증가했습니다."
- "대전 소상공인과 제휴가 완료됐습니다."
- "AI가 실제 방문·매출 데이터를 분석했습니다."
- "타슈 잔여 대수가 실시간으로 연동됩니다."

## 상용 전환 게이트

1. 가게별 정보 출처와 점주 확인 상태를 관리합니다.
2. 이미지별 저작권·사용 범위·만료일을 메타데이터로 저장합니다.
3. 코스 정류장의 영업시간, 휴무일, 이동 가능성을 주기적으로 검증합니다.
4. 분석 화면의 모든 수치에 집계 기간, 출처, 정의, 표본 크기를 표시합니다.
5. 개인 위치·방문 데이터는 최소 수집, 보유 기간, 파기, 익명화 정책을 적용합니다.

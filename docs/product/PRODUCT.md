# 제품 정의와 현재 범위

## 한 문장 정의

**놀거많은대?전은 대전의 장소를 이름이 아닌 이야기와 동선으로 발견하게 하고, 사용자의 한 곳 방문을 주변 상권의 여러 경험으로 확장하는 로컬 탐방 서비스입니다.**

## 핵심 타깃

| 고객 | 해결할 일 | 제품 가치 |
| --- | --- | --- |
| 대전 시민·방문객 | 오늘 가고 싶은 곳과 그 다음 일정을 빠르게 결정 | 검색·상황형 코스·ETA 동선 |
| 대전 소상공인 | 가게의 차별점을 설명하고 실제 방문으로 연결 | 스토리 상세, 코스 편입, 유입 신호 |
| 상인회·지자체·관광기관 | 특정 명소에 집중된 방문을 주변 상권으로 분산 | 테마 코스, 캠페인, 집계 리포트 |

## 차별화 가설

일반 지도·플레이스 서비스가 "어디에 있는지"를 효율적으로 보여준다면, 놀거많은대?전은 "왜 가야 하는지"와 "간 뒤에 무엇을 할지"를 연결합니다. 핵심 차별화는 점주 서사, 주변 가게와의 코스 연결, 실행 가능한 이동 계획을 하나의 흐름으로 제공하는 데 있습니다.

## 현재 기능 상태

| 기능 | 상태 | 비고 |
| --- | --- | --- |
| 브랜드 Splash·온보딩 | 구현 | 테마·연령대 선택 |
| 통합 검색 | 구현 | 이름, 카테고리, 지역, 설명, 태그, 메뉴 |
| 검색 + 카테고리 | 구현 | 두 조건을 `AND`로 적용, 빈 결과 처리 |
| 대전 지도 탐색 | 구현 | 장소·타슈 표시, 지도 스타일, 위치 행동 |
| 코스·ETA 재계산 | 구현 | MapKit 도로 기준, 도보·타슈 모드 |
| 가게 스토리 상세 | 구현 | 램프의진희 실자료 적용 |
| 저장·방문·보상 | 로컬 구현 | 계정 없이 UserDefaults에 영속 |
| 공유 경로 피드 | 프로토타입 | 5개 예시 경로. 생성·게시 백엔드 미구현 |
| 타슈 재고 | 프로토타입 | 8개 정적 예시 거점, 실시간 API 미연동 |
| 사장님 분석 | 프로토타입 | 결정적 목 데이터. 실제 AI·POS·방문 집계가 아님 |
| Supabase | 설계 | SQL 스키마·RLS 초안만 존재 |

## 핵심 검증 질문

| 가설 | 최소 검증 방법 | 판정 지표 |
| --- | --- | --- |
| 스토리 콘텐츠가 단순 리스트보다 방문 의사를 높인다. | 동일 가게의 일반 카드와 스토리 카드 A/B 비교 | 상세 진입률, 저장률, 지도 행동률 |
| 한 유명 장소 이후 코스가 주변 상권 방문을 늘린다. | 테스트 코스 시작 전·후 지오펜스 방문 기록 | 코스 시작률, 2번째 정류장 도달률, 완주율 |
| 소상공인이 노출·효과 데이터에 대해 지불할 의사가 있다. | 10개 점포 무료 파일럿 후 유료 옵션 인터뷰 | 콘텐츠 검수 소요시간, 리포트 열람률, 유료 전환 의향 |
| 상인회·지자체가 상권 분산 지표를 활용할 수 있다. | 행사 또는 코스 1건의 익명 집계 리포트 제공 | 방문 분산도, 평균 정류장 수, 관련 상권 체류 시간 |

## 현실적인 다음 단계

1. 램프의진희를 포함한 대흥동 5개 점포의 콘텐츠 동의·검수 파일럿을 완료합니다.
2. 앱 이벤트 스키마를 확정하고 조회, 저장, 코스 시작, 지도 클릭, 방문 인증을 수집합니다.
3. Supabase로 콘텐츠·사용자 상태를 전환하고 점주별 권한을 분리합니다.
4. 30명 내외의 현장 테스트로 탐색 시간, 코스 시작률, 2개 이상 방문률을 검증합니다.
5. 지불 전환은 광고비부터 요청하지 않고, 무료 입점 효과를 확인한 점포를 대상으로 선택형 프로모션·분석 요금제를 검증합니다.

# 「놀거많은대?전」 발표 PPT 설계 마스터 프롬프트

## 1. 분석 결론

### 참고 PPT에서 유지할 흐름

- 첫 문장을 질문으로 시작해 문제의 긴장감을 즉시 만든다.
- 관광 수요 증가와 원도심 상권 불균형을 연결한다.
- 기존 지도·SNS 소개 방식의 한계를 설명한 뒤 해결안을 제시한다.
- 앱 기능을 실제 화면으로 보여주고, 참여 가게의 부담과 기대효과를 설명한다.
- 마지막에는 상인회·지역기관에 필요한 협력 사항을 명확히 요청한다.

### 새 발표에서 반드시 보완할 부분

- 기능을 나열하지 말고 `문제 → 사용자 행동 → 앱 개입 → 가게 방문 확산`의 인과관계로 설명한다.
- 실제 점주 제공 자료를 앱에 반영한 `램프의진희` 사례를 핵심 검증 근거로 사용한다.
- 네이버지도·카카오맵·대한민국 구석구석·대전관광과의 차이를 기능 우열이 아닌 역할 차이로 비교한다.
- 현재 앱 화면은 파랑·청록 색상으로 개편됐으므로 참고 PDF의 예전 주황색 앱 이미지를 최종 발표에 재사용하지 않는다.
- 사업모델은 초기 무료 참여와 이후 유료 전환을 분리해, 소상공인에게 처음부터 광고비를 요구하는 인상을 주지 않는다.
- 사장님 대시보드의 매출·방문 수치는 실제 성과가 아니라 `분석 화면 프로토타입·예시 데이터`임을 표시한다.
- 다른 가게 예시 데이터와 실제 점주 제공 자료를 구분한다. `램프의진희`는 실제 제공 자료 기반, 그 외 일부 가게는 MVP 시나리오용 예시 데이터라고 투명하게 표기한다.
- 출처가 확인되지 않은 방문객 수나 시장 규모 수치를 확정적으로 쓰지 않는다. 모든 수치는 원문 기준·단위·기간을 확인한 뒤 각주를 붙인다.

### 권장 발표 방식

세 가지 방식 중 `데이터 + 현장 사례 + 제품` 하이브리드 구성을 사용한다.

1. 데이터 중심형: 심사 대응에는 강하지만 서비스의 감정적 매력이 약하다.
2. 가게 다큐멘터리형: 기억에는 남지만 사업성과 확장 논리가 약해질 수 있다.
3. 하이브리드형: 문제를 데이터로 증명하고, 실제 가게 사례로 신뢰를 만들며, 제품과 사업모델로 실행 가능성을 보여준다.

## 2. PPT 제작용 최종 마스터 프롬프트

아래 내용을 그대로 Canva, Gamma, Figma Slides, PowerPoint 제작 AI 또는 발표자료 제작자에게 전달한다.

```text
당신은 지역상권·관광·스타트업 피치덱을 전문으로 만드는 프레젠테이션 전략가이자 아트디렉터다.

대전 지역 기반 로컬 탐방 앱 「놀거많은대?전」의 16:9 발표자료를 제작하라. 발표 대상은 상인회, 지역기관 관계자, 창업지원 심사위원이다. 기본 발표 시간은 약 10분이며, 본편 18장과 부록 3장으로 구성한다.

[발표의 커뮤니케이션 목표]
발표가 끝났을 때 청중은 「놀거많은대?전」이 단순한 맛집·관광 앱이 아니라, 유명 거점에 집중된 방문객을 주변 골목과 로컬 가게로 연결하는 지역상권 탐방 인프라이며, 실제 점주 제공 자료를 앱 콘텐츠로 전환할 수 있는 실행 단계의 서비스라고 이해해야 한다. 최종적으로 원도심 실증을 위한 참여 가게 연결, 현장 정보 제공, 사용자 테스트 및 협업 지원에 동의하도록 설득한다.

[한 문장 정의]
놀거많은대?전은 대전의 가게·관광지·축제·카페·체험 공간을 점주의 이야기, 사용자 취향, 도보·타슈 이동 동선으로 연결해 “한 곳 방문”을 “동네 전체 경험”으로 바꾸는 지역 기반 추천 앱이다.

[핵심 메시지]
문제는 좋은 가게가 없는 것이 아니라, 좋은 가게가 발견되고 다음 방문으로 연결되는 구조가 부족하다는 것이다.

[발표 전체 서사]
방문객은 늘지만 골목까지 흐르지 않는다 → 기존 서비스는 위치와 리뷰는 잘 보여주지만 지역 맥락과 다음 동선을 충분히 연결하지 못한다 → 놀거많은대?전은 점주 서사와 사용자 상황을 코스로 묶는다 → 실제 소상공인 ‘램프의진희’ 자료를 상세페이지와 주변 코스로 구현했다 → 초기 무료 실증으로 참여 장벽을 낮추고, 검증 이후 B2C·B2B·B2G 수익모델로 확장한다 → 대전 원도심에서 효과를 측정한 뒤 대전 전역과 다른 지역으로 확장한다.

[시각 콘셉트]
- 콘셉트명: 도시를 연결하는 파란 길, 그 안에서 발견되는 따뜻한 가게
- 스타일: 공공기관 보고서처럼 딱딱하지 않고, 스타트업 피치덱처럼 과장되지 않은 에디토리얼 다큐멘터리 스타일
- 화면 비율: 16:9, 1920×1080 기준
- 기본 색상: Navy #061B52, Blue #2E6BFF, Cyan #11B8D6, Signal Green #32E982, Off-white #F7FBFF, Text #08192F
- 앱의 파랑·청록 색상을 기본으로 사용하고, 실제 가게 사진의 초록·주황·수공예 질감을 포인트로 활용한다.
- 제목 폰트는 Gmarket Sans Bold 또는 Noto Sans KR Black, 본문은 Pretendard 또는 Noto Sans KR을 사용한다.
- 표지 제목 54pt 이상, 슬라이드 제목 38~44pt, 중간 제목 24~28pt, 본문 18~22pt, 출처 10~12pt를 기준으로 한다.
- 슬라이드마다 하나의 주장만 전달한다. 제목은 ‘프로젝트 배경’ 같은 분류명이 아니라 결론을 말하는 문장으로 작성한다.
- 텍스트 25%, 이미지·차트·앱 화면 75%를 목표로 한다.
- 일반적인 카드 UI를 반복하지 말고, 큰 사진·큰 숫자·간결한 도식·여백 중심으로 구성한다.
- 파란 경로선이 장소와 앱 화면을 연결하는 시각 모티프를 여러 장에서 일관되게 사용한다.
- 한 슬라이드에 휴대폰 화면은 최대 3개까지만 배치하고, 핵심 부분을 확대하거나 번호 콜아웃으로 설명한다.
- 실제 가게 사진이 있는 경우 일반 스톡 이미지를 사용하지 않는다.

[이미지 및 근거 사용 원칙]
- ‘램프의진희’ 실제 사진 8장을 적극 활용한다: 매장 전경, 작품 선반, 말 그림, 캐릭터 작품, 패브릭 작업, 협업 패키지, 문화거리 전시, 손글씨 소개문.
- 앱 화면은 현재 빌드에서 새로 캡처한다. 참고 PDF에 들어 있는 과거 주황색 앱 화면은 사용하지 않는다.
- 비교 서비스 이미지는 공식 서비스 페이지 또는 공식 App Store 화면만 사용하고, 이미지 아래에 작은 출처를 표시한다.
- 통계는 제공된 원도심 분석보고서의 원문을 재확인한 후 사용한다. 기준·단위·기간이 불분명하면 숫자를 삭제하고 정성 메시지로 대체한다.
- ‘대전 방문객 9천만 명’, ‘성심당 방문객 1천만 명’ 같은 수치는 공식 원문을 확인하기 전에는 본문에 넣지 않는다.
- 사장님 대시보드 숫자는 ‘예시 데이터 기반 프로토타입’이라고 명시한다.
- 실제 점주 협업 사례와 MVP 예시 데이터를 시각적으로 구분한다.

[슬라이드별 설계]

1. 표지 — “숨은 가게의 이야기를, 대전의 새로운 즐길 거리로”
목적: 서비스의 정체성을 한 문장으로 각인한다.
구성: 왼쪽에 브랜드명 「놀거많은대?전」과 부제, 오른쪽에는 대전 원도심 지도 실루엣 위로 성심당·은행동·대흥동·선화동을 연결하는 파란 경로선과 실제 ‘램프의진희’ 사진 2~3장을 콜라주한다.
표시 문구: “대전 로컬 가게와 관광 경험을 연결하는 스토리·코스 기반 탐방 앱”

2. 오프닝 질문 — “방문객은 늘어나는데, 왜 골목 가게는 사라질까?”
목적: 참고 PPT의 강한 질문형 도입을 유지한다.
구성: 질문을 화면 중앙에 크게 배치하고, 한쪽에는 붐비는 대표 거점, 다른 쪽에는 조용한 골목의 대비 이미지를 배치한다. 설명은 한 문장만 둔다.
보조 문구: “같은 원도심 안에서도 방문 흐름은 특정 지점에 멈춥니다.”

3. 문제의 본질 — “가게 수가 아니라, 발견되는 구조의 차이가 매출과 생존을 가른다”
목적: 문제를 정량 근거로 제시한다.
구성: 참고 PPT의 원도심 월평균 매출·상권 수·폐업 건수 차트를 단순화해 재제작한다. 대흥동은 상점 수가 많아도 매출이 상대적으로 낮고 폐업이 더 많다는 한 가지 결론만 강조한다.
주의: 26.5/22.1, 490/613, 70/107 등의 수치는 원문 보고서에서 단위·권역·기간을 재확인한 경우에만 사용한다.
각주: “출처: 2025년 11월 원도심 상권 분석보고서, 세부 기준 재확인 후 표기”

4. 기존 탐색 방식 — “기존 서비스는 ‘어디 있는지’는 알려주지만, ‘왜 가야 하는지’와 ‘다음 어디로 갈지’는 흩어져 있다”
목적: 경쟁 서비스를 비판하지 않고 미충족 수요를 보여준다.
구성: 네이버지도, 카카오맵, 대한민국 구석구석, 대전관광의 공식 화면을 4분할로 배치한다. 각 서비스의 강점을 먼저 적고, 아래에 놀거많은대?전이 보완하는 지점을 적는다.
비교 문구:
- 네이버지도·카카오맵: 검색, 리뷰, 저장, 길찾기에 강함
- 대한민국 구석구석: 전국 관광정보와 추천 코스에 강함
- 대전관광: 공식 지역 관광·가이드북 정보에 강함
- 놀거많은대?전: 점주 서사, 골목 단위 연결, 도보·타슈 코스, 참여 가게 효과 측정에 집중

5. 비교 포지셔닝 — “우리는 지도 앱을 대체하지 않고, 지도 위에 ‘방문할 이유와 흐름’을 더한다”
목적: 서비스의 역할을 명확히 한다.
구성: 4열 비교표를 간결하게 만든다. 열은 범용 지도 / SNS·블로그 / 공공 관광정보 / 놀거많은대?전, 행은 정보 단위 / 이야기 깊이 / 동선 연결 / 소상공인 부담 / 지역 유입 분산 목표다.
핵심 강조: 놀거많은대?전의 정보 단위는 개별 장소가 아니라 ‘동네 경험’이다.

6. 사용자 순간 — “성심당 다음에는 어디 가지?”
목적: 실제 사용 상황으로 문제를 구체화한다.
구성: 대전 방문객의 90분 시나리오를 한 줄 타임라인으로 보여준다.
시나리오: 성심당 방문 → 검색 피로 → 앱에서 ‘대흥동 그림 소품 산책’ 발견 → 도보·타슈 이동 → 로컬 가게 2~3곳 방문 → 코스 저장·공유.
카피: “유명한 한 곳에서 끝나는 방문을, 기억에 남는 동네 경험으로 바꿉니다.”

7. 해결안 한눈에 보기 — “점주의 이야기와 사용자의 상황을 하나의 코스로 연결한다”
목적: 서비스 구조를 10초 안에 이해시킨다.
구성: `점주 인터뷰·사진 → 스토리 상세페이지 → 취향·시간 검색 → 도보·타슈 코스 → 방문·저장·공유 → 상권 인사이트`의 6단계 흐름을 하나의 파란 선으로 연결한다.
중앙에는 앱 홈 화면을 크게 배치한다.

8. 홈·검색 — “대전에서 뭐 할지, 장소명이 아니라 상황으로 찾는다”
목적: 현재 구현된 검색과 카테고리 필터를 보여준다.
사용 화면: 홈 기본 화면, ‘성심당’ 검색 결과, ‘체험’ 또는 ‘데이트’ 카테고리 필터 화면.
콜아웃: 장소명·카테고리·지역·설명·태그 통합검색 / 검색어 AND 카테고리 필터 / 결과 없음 안내 / 상세페이지 연결.
보조 문구: “맛집, 카페, 관광지, 축제, 데이트, 가족, 체험, 야간명소를 한 화면에서 탐색”

9. 코스·지도 — “추천은 장소 목록이 아니라 실제로 움직일 수 있는 순서여야 한다”
목적: 핵심 차별 기능인 이동 경험을 설명한다.
사용 화면: 코스 상세 상단, 코스 순서, 지도 경로, 도보·타슈 예상 시간.
콜아웃: 체류시간 포함 / 도보·타슈 선택 / ETA 기준 추천 순서 / 지도에서 한눈에 보기 / 초행자도 따라가기 쉬운 동선.

10. 스토리 상세페이지 — “평점 대신, 이 가게를 찾아갈 이유를 보여준다”
목적: 일반 장소 상세와의 차이를 시각적으로 전달한다.
사용 화면: 가게 상세 히어로, ‘왜 이 가게를 시작했나요?’, ‘이 가게만의 특별함’, 메뉴·서비스, 공간 갤러리, 방문 팁.
구성: 한 화면 전체를 작게 보여주지 말고, 핵심 섹션 3개를 확대해 배치한다.

11. 현장 사례 1 — “실제 점주가 준 자료는 이미 앱의 콘텐츠가 되었다”
목적: 아이디어가 아닌 실행 근거를 보여준다.
사례: 대흥동 문화거리의 ‘램프의진희’.
사실 정보:
- 2012년 8월 16일 문을 열어 2026년 현재 14년째 운영
- 그림 전시·판매, 그림 공방, 개성 있는 소품을 함께 운영
- ‘램프’는 소망을 이루어주는 존재, ‘진희’는 작가의 이름과 참됨·진정성의 의미를 연결
- “일상 속 작은 빛이 되어 마음의 불을 켜고 그리다”라는 공간 철학
구성: 점주가 제공한 매장·작품 사진 6~8장을 잡지 화보처럼 배치하고, 손글씨 소개문을 크게 사용한다.
표기: “실제 점주 제공 자료 기반”

목적: 콘텐츠 생산 과정을 증명한다.
구성: 왼쪽에는 점주 제공 원자료, 중앙에는 구조화된 필드, 오른쪽에는 실제 앱 상세페이지와 코스를 배치한다.

2) 약 20분 인터뷰와 사진 제공

사용자 가치: 상황형 추천, 실패 비용 감소, 이동 편의, 새로운 장소 발견.

지역 가치: 방문 분산, 체류시간 증가, 로컬 콘텐츠 축적, 축제·정책 연계.

목적: 수익화 시점과 고객을 현실적으로 설명한다.
구성: 3단계 로드맵과 B2C·B2B·B2G 모델을 연결한다.

- B2C: 핵심 기능 무료, 프리미엄 큐레이션·개인화·광고 제거·시즌 패스

- B2G: 지자체·관광기관·축제 운영사 대상 공식 테마 코스 제작, 캠페인 운영, 익명·집계 기반 상권 인사이트 리포트
```
